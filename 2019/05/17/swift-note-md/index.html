Swift学习笔记

发表于 2019-05-17 |

基础

  • Optional 可以用在所有的类型上

  • typealias AudioSample = UInt16 别名

  • 合并空值运算符 a ?? b， a必须是optional，等价为a != nil ? a! : b

  • String可以用 + 运算符，可以用 == 比较

  • Array也可以用 + 运算符

  • String, Int, Double, Bool 都是可哈希的

  • do while 叫在swift叫 repeat while

Switch

  • switch 不需要显示break，一个情况匹配多个值用逗号隔开 case 'a', 'A':

  • 可以区间匹配 case 1..<5:

  • 可以匹配元组，并可用 _ 来匹配所有值

  • 可以用值绑定

  • 可以用where来检查额外情况 case let (x, y) where x == y

函数

  • 支持默认参数 func someFunction(parameterWithDefault: Int = 12)
  • 可变形参 func arithmeticMean(_ numbers: Double...) -> Double，在函数体内会被当做名字是numbers的[Double]常量数组
  • inout来改变参数 `func swap(_ a: inout Int, _ b: inout Int)

####

闭包

  • 函数和闭包都是引用类型
  • @escaping 逃逸闭包，如果它作为参数传递进去，需要函数返回之后还能被调用就需要逃避闭包，通常作用异步调用block。逃逸闭包内必须显示self。
  • @autoclosure 自动闭包，在参数用@autoclosure修饰时，传递参数可以像普通参数一样传递省略掉闭包语法的大括号。

枚举

  • switch检查枚举时应该为枚举的所有情况都覆盖，当然也可以提供default

  • 遍历枚举。可以通过在枚举名字后面加 CaseIterable ，swift会提供allCases这个集合。

  • 关联值。 关联值只是指定类型

  • 原始值。 原始值指定了具体值，需要每个值类型都一样

    隐士指定原始值。整数时，第一个默认为0，之后递增1。字符串时，原始值默认为它的名称。可以使用rawValue访问原始值。

  • 枚举递归，使用indirect来表示可以递归

类和结构体

类和结构体共同之处：

  • 可以定义属性
  • 可以定义方法
  • 可以定义下标访问
  • 可以定义初始化器
  • 可以用extension
  • 可以遵循协议

不同之处：

  • 类可以继承
  • 类有多态
  • 类可以在析构时释放分配的资源
  • 结构体是值类型，类是引用
  • 结构体可以接受默认的成员初始化器
  • let修饰的结构体，即便属性时var，也不能修改。但类的可以。

其他注意事项：

  • 判断两个变量或者常量是否引用同一个实例，可以用 === 或者 !== 进行判断
  • lazy 修饰的延迟存储属性，必须用var修饰，因为可能是在类初始化后才进行赋值的

类的初始化器规则

  • 指定初始化器必须从它的直系父类调用指定初始化器
  • 便捷初始化器必须从相同的类里调用另一个初始化器
  • 便捷初始化器最终必须调用一个指定初始化器

总结：指定初始化器必须总是向上委托。便捷初始化器总是横向委托。

类的初始化两阶段

阶段一
  • 指定或便捷初始化器在类中被调用；
  • 为这个类的新实例分配内存。内存还没有被初始化；
  • 这个类的指定初始化器确保所有由此类引入的存储属性都有一个值。现在这些存储属性的内存被初始化了；
  • 指定初始化器上交父类的初始化器为其存储属性执行相同的任务；
  • 这个调用父类初始化器的过程将沿着初始化器链一直向上进行，直到到达初始化器链的最顶部；
  • 一旦达了初始化器链的最顶部，在链顶部的类确保所有的存储属性都有一个值，此实例的内存被认为完全初始化了，此时第一阶段完成。

阶段二

  • 从顶部初始化器往下，链中的每一个指定初始化器都有机会进一步定制实例。初始化器现在能够访问 self 并且可以修改它的属性，调用它的实例方法等等；
  • 最终，链中任何便捷初始化器都有机会定制实例以及使用 slef 。

类的初始化安全检查

安全检查1：

指定初始化器必须保证在向上委托给父类初始化器前，其所在类引入的所有属性都要初始化完成

安全检查2：

指定初始化器必须向上委托父类初始化器，然后才能为继承的属性设置新值。否则新值会被父类的初始化器覆盖

安全检查3：

便捷初始化器必须先委托同类中的其他初始化器，然后再为任意属性赋新值。否则新值会被同类的初始化器覆盖

安全检查4：

初始化器在第一阶段初始化完成前，不能调用任何实例方法，不能读取任何实例属性的值，也不能引用self作为值

初始化器的继承和重写

Swift的子类不会默认继承父类的初始化器。

重写父类的指定初始化器需要加override，即便子类中它是一个便捷初始化器。重写父类的便捷初始化器不需要加。

默认继承初始化器的规则
  • 如果子类没有定义任何指定初始化器，会自动继承所有父类的初始化器
  • 如果子类提供了所有父类的指定初始化器的实现 — 要么是通过上述规则实现，要么自定义实现的。那么它自动继承所有父类的便捷初始化器

必要初始化器

使用require关键字表明所有该类的子类都必须实现该初始化器
