iOS底层原理 - OC 对象

发表于 2019-05-03 |

OC 对象分类

话不多说，直接给答案：

  1. 实例对象
  2. 类对象
  3. 元类对象
实例对象

实例对象就是通过类 alloc 出来的对象，每次 alloc 都会产生新的实例对象。

实例对象在内存中存储信息为：

  1. isa 指针
  2. 其他成员变量

那么实例对象方法存储在什么地方呢？答案是存放在类对象内存中，而类对象的地址就是实例对象中isa指向的地址。也就是每个实例对象的isa都指向的是其类对象的地址。

类对象

我们可以通过实例对象的class方法或者object_getClass(object)得到它的类对象，每一个类在内存中只有一份类对象，而类对象的存储信息为：

  1. isa指针
  2. superclass指针
  3. 类的属性信息（@property），类的成员变量信息（ivar）
  4. 类的实例方法，类的协议信息

这里读者肯定有疑问，上面不是说类的成员变量存储在实例对象中么，这里怎么又说在类对象中？其实是成员变量的值存放在实例对象中，而成员变量叫什么名字、什么类型只需一份就够了，所以存放在类对象中。

那么类方法在哪里？答案是在元类对象里

元类对象

元类对象和类对象的内存结构是一样的，但是用途不一样，其内存信息为：

  1. isa指针
  2. superclass指针
  3. 类方法

总结下来，可以通过对象的isa指针找到它的类对象，通过类对象的isa找到其元类对象，这样便找到某对象的实例方法和类方法完成调用。而元类的isa指针，指向元类自身。
